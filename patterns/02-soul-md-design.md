# Pattern 2: SOUL.md = 80% of Agent Quality

**Problem:** Agents with vague instructions produce vague output.

**Solution:** SOUL.md is the agent's job description — specific mission, exact sources, output format, rules.

**Good SOUL.md test:** If you gave this to a new hire on day 1, would they know exactly what to do? If not, add more detail.

**Must include:**
- Mission (1-2 specific sentences)
- Sources to check (with URLs)
- Output format (exact template)
- Team @mentions (bot IDs)
- Channel IDs
- Rules and constraints

**Spend 80% of your agent design time on SOUL.md.**
