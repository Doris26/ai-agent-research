### Q1: Google Antigravity — ADK/Agent Engine architecture

**Verdict: Antigravity is the IDE layer; ADK+Agent Engine are the framework+runtime layers. They are complementary, not overlapping.**

Key findings:
- Antigravity is built on a VS Code fork, NOT on ADK/Agent Engine primitives directly
- However, it has explicit ADK integration: Google published "Creating an ADK Agent Skill in Antigravity" (Medium) and "Spec-Driven ADK Agent Development with Antigravity and Spec-kit" (Codelabs)
- ADK skills run in Antigravity's skill system — you build ADK agents *using* Antigravity as the IDE
- For deployment: the "Build and Deploy to GCP with Antigravity" codelab shows Cloud Run + BigQuery + Pub/Sub deployment — NOT direct `adk deploy` to Agent Engine from within the IDE
- `adk deploy` to Agent Engine remains a CLI command you run separately; Antigravity doesn't wrap it yet
- GCP native bias: Antigravity deeply integrates with GCP services (Cloud Run, Firebase, BigQuery) but is multi-model (Claude Sonnet/Opus, GPT-OSS, Gemini 3)

**GCP moat verdict:** Partial. The GCP service integration (deploy to Cloud Run, Data Cloud) is a real pull factor, but the multi-model stance and VS Code fork architecture mean competitors can replicate. The ADK+Antigravity workflow is a developer productivity moat, not a lock-in moat. Agent Engine remains the production runtime — Antigravity just accelerates building the agents that run there.

### Q2: Alibaba Wukong — Developer surface, MCP compatibility, tool registry

**Verdict: MCP-native confirmed. Has a planned B2B skill marketplace (not yet GCP-equivalent). SDK coming but not yet public.**

Key findings:
- **MCP: YES** — confirmed MCP native support; connects to external services via MCP protocol
- **Skill Marketplace:** Alibaba announcing a dedicated AI Skill marketplace for Wukong — B2B, supports full skill lifecycle (creation → review → distribution), "compatible with open-source skill frameworks". Ambition: "world's largest B2B skill marketplace"
- **Developer API surface (command-level):** `search_skills`, `use_skill`, `cli_skills_install_local/url`, `browser_start/screenshot`, `execute_shell`, `cron_*` ops — exposed as agent-callable commands, not REST API
- **No public REST API or SDK docs yet** — still invitation-only beta; SDK + API ecosystem "announced" but not documented
- **Technical stack:** Tauri 2.x + Rust desktop app, embedded Bun/Node/Python/Chromium runtimes, Playwright for browser automation, model-agnostic (Claude, Gemini, OpenAI, local)

**vs GCP ApiRegistry:**
- GCP ApiRegistry: admin-governed, org-controlled, curated tool registry — top-down governance
- Wukong Skill Marketplace: open B2B app-store model — bottom-up, ISV-driven, community distribution
- These are different models. GCP is missing the open marketplace distribution layer. Wukong's marketplace is more like an Apple App Store for enterprise agents; GCP's ApiRegistry is more like an internal service catalog.
- **GCP gap:** No open skill/agent marketplace for third-party ISVs to publish and distribute agent capabilities on Agent Engine.
