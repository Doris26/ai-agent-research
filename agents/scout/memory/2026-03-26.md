### FINDING 1: Cursor Subagent Architecture

**Source URLs:**
- https://cursor.com/docs/subagents
- https://cursor.com/changelog/2-4
- https://cursor.com/blog/self-hosted-cloud-agents
- https://www.startuphub.ai/ai-news/technology/2026/cursor-lets-ai-agents-run-on-your-infrastructure

**Technical Architecture:**
- Subagents are independent agents with their own isolated context windows
- Run in TWO modes: **Foreground** (blocks until done) and **Background** (async, returns immediately)
- Up to **8 agents simultaneously**, each in isolated git worktrees (separate codebase copies)
- Three built-in subagents: **Explore** (codebase nav, fast model), **Bash** (shell execution, isolated), **Browser** (DOM-filtered web automation)
- Custom subagents defined as markdown files (SKILL.md format) in project or user directories
- Subagents can spawn further subagents (recursive nesting) — complex workflows don't bottleneck

**Self-Hosted Cloud Agents (new, Mar 25):**
- Worker model: each worker is a process connecting **outbound HTTPS** to Cursor's cloud — no inbound ports, no VPN, no firewall changes
- Split execution: Cursor cloud handles AI inference + planning; worker handles tool execution on customer hardware
- Kubernetes deployment: Helm chart + Kubernetes operator for pool scaling and lifecycle management
- Fleet management API for non-K8s environments (autoscaling, utilization monitoring)
- Command: `agent worker start` — single-task or long-lived process
- Agents launch from Cursor editor, Slack, or GitHub
- 30+ partner plugins (Atlassian, Datadog, GitLab, Glean, Hugging Face, PlanetScale, monday.com)
- Enterprise customers: **Brex** (internal test suites), **Money Forward** (financial compliance), **Notion** (avoids multi-stack maintenance)

**No Public API:** There is no external API for third-party developers to integrate. The platform is Cursor-proprietary — custom subagents work inside Cursor only.

**Platform vs Product Signal:**
Cursor is now clearly a **managed agent platform**: customer owns infrastructure/code/secrets, Cursor owns orchestration/inference/models. This is the same pattern AWS is doing with Bedrock AgentCore. Cursor has the IDE moat plus the enterprise infrastructure moat.

**GCP Gap:**
- Antigravity (Google's agentic IDE) has no equivalent subagent architecture published
- ADK has no multi-agent parallel IDE-native execution model
- No self-hosted worker runtime from GCP that keeps code on-premise while outsourcing orchestration
- Cursor's plugin ecosystem (30+ partners) has no equivalent ADK marketplace

### FINDING 2: Hindsight Reflection Loops — Technical Deep Dive

**Source URLs:**
- https://github.com/vectorize-io/hindsight
- https://hindsight.vectorize.io/
- https://hindsight.vectorize.io/blog/2026/03/24/langgraph-longterm-memory
- https://arxiv.org/html/2512.12818v1
- https://venturebeat.com/data/with-91-accuracy-open-source-hindsight-agentic-memory-provides-20-20-vision

**Three Core Memory Networks:**
1. **World** — objective facts about the world
2. **Experiences** — agent's own interaction history
3. **Mental Models / Opinions** — learned beliefs formed by reflecting on raw facts

**Three Core Operations:**
- `retain()` — stores interactions, extracts structured facts with timestamps and entity links
- `recall()` — retrieves using TEMPR (4 parallel strategies)
- `reflect()` — synthesizes across memories; the self-improvement loop

**TEMPR Retrieval (4 parallel strategies run simultaneously):**
1. Semantic vector similarity (conceptual matching)
2. BM25 keyword matching (exact terms, technical names)
3. Graph traversal (entity relationships, indirect connections)
4. Temporal filtering (time-constrained queries: "last spring", date ranges)
All four results are merged and re-ranked with a cross-encoder.

**reflect() Algorithm (formal definition: Reflect(B,Q,Θ)→(r,𝒪')):**
1. TEMPR retrieval fetches relevant facts from all four memory networks
2. Profile loading: reads behavioral parameters (skepticism S, literalism L, empathy E, all 1-5 scale)
3. Preference-conditioned generation: LLM produces response shaped by disposition + bias-strength β ∈ [0,1]
4. Opinion formation: new beliefs created with confidence scores
5. Memory update: new opinions stored for future reinforcement

**Opinion Reinforcement (how beliefs evolve over time):**
- Finds candidate opinions via entity overlap + semantic similarity
- Classifies new facts as: reinforce, weaken, contradict, or neutral
- Confidence score update formula: `c' = min(c+α, 1.0)` for reinforce; `max(c-α, 0.0)` for weaken; `max(c-2α, 0.0)` for contradiction

**CARA (Coherent Adaptive Reasoning Agents):**
- Disposition parameters verbalized into natural language, injected into system messages
- Skepticism (S): 1=trusting → 5=highly skeptical evaluation
- Literalism (L): 1=flexible interpretation → 5=strictly literal
- Empathy (E): 1=detached → 5=empathetic reasoning
- Same facts can yield different conclusions based on configured disposition

**LangGraph Integration (3 patterns):**
1. **Tools Pattern**: Hindsight exposed as 3 LangChain tools (`hindsight_retain`, `hindsight_recall`, `hindsight_reflect`) — agents call autonomously
2. **Nodes Pattern**: Memory operations as dedicated graph steps (recall before LLM, retain after); bank IDs resolve from runtime config for per-user isolation
3. **BaseStore Pattern**: Replaces LangGraph's `InMemoryStore` as the storage backend

**Per-user isolation code:**
```python
recall = create_recall_node(client=client, bank_id_from_config="user_id")
# config={"configurable": {"user_id": "user-456"}}
```

**Deployment:** Docker Compose, Helm, pip, or cloud (ui.hindsight.vectorize.io). Python, TypeScript, Go, CLI, HTTP API.

**Benchmark Results:**
- LongMemEval: 91.4% (Hindsight + Gemini-3) vs 60.2% (full-context GPT-4o baseline)
- LoCoMo: 89.61% vs 75.78% previous SOTA

**Comparison to Other Memory Systems:**
| Feature | Zep | A-Mem | Mem0 | Hindsight |
|---------|-----|-------|------|-----------|
| Separates facts/opinions | ✗ | ✗ | ✗ | ✓ |
| Temporal reasoning | ✓ | ✗ | ✗ | ✓ |
| Opinion evolution | ✗ | ✗ | ✗ | ✓ |
| Behavioral parameters | ✗ | ✗ | ✗ | ✓ |
| Confidence scores | ✗ | ✗ | ✗ | ✓ |

**Vertex AI Memory Bank Gap Analysis:**
- Memory Bank does: simple retrieval, similarity search, retrospective reflection (extracts facts from past conversations, consolidates into evolving memories, async background processing)
- Memory Bank does NOT: 4-strategy parallel retrieval, opinion/fact separation, confidence score tracking, belief evolution (reinforce/weaken/contradict), behavioral disposition parameters, entity knowledge graph, TEMPR temporal reasoning
- Memory Bank gap: pure storage + retrieval, no belief formation, no self-improving reasoning layer
- Hindsight can be deployed on top of any LangChain/LangGraph agent — could run alongside Vertex AI ADK agents as an external memory layer
- Paper reference: arXiv:2512.12818 — "Hindsight is 20/20: Building Agent Memory that Retains, Recalls, and Reflects"
