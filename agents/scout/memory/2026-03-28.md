### Cloud Provider Updates (AWS)

1. **AWS Bedrock AgentCore — Managed Session Storage (Preview)**
   - URL: https://aws.amazon.com/about-aws/whats-new/2026/03/bedrock-agentcore-runtime-session-storage/
   - Agents now persist filesystem state across stop/resume cycles via managed session storage
   - Everything agent writes to configured mount path persists across microVM restarts — source files, packages, build artifacts, git history all intact
   - Resume via same session ID → new microVM mounts same storage
   - GCP equivalent: None — Vertex AI Agent Engine has Sessions GA but no persistent filesystem mounts
   - Gap: GCP agents lose state on compute restarts; AWS agents persist full filesystem
   - Why it matters: Eliminates one of the biggest pain points in long-running agents (context loss across restarts)

2. **AWS Bedrock AgentCore — AG-UI Protocol Support**
   - URL: https://aws.amazon.com/about-aws/whats-new/2026/03/amazon-bedrock-agentcore-runtime-ag-ui-protocol/
   - AgentCore now supports Agent-User Interaction (AG-UI) open event-based protocol
   - AG-UI standardizes how AI agents communicate with user interfaces (complements MCP + A2A)
   - Triad now complete: MCP (tools) + A2A (agent-to-agent) + AG-UI (agent-to-user)
   - GCP equivalent: No equivalent protocol support announced
   - Gap: GCP Agent Engine has no standardized protocol for real-time UI rendering from agents
   - Why it matters: AWS is building a complete protocol stack; GCP has Cloud API Registry for MCP only

3. **AWS Bedrock AgentCore — Stateful MCP Server Features (confirmed GA)**
   - URL: https://aws.amazon.com/about-aws/whats-new/2026/03/amazon-bedrock-agentcore-runtime-stateful-mcp/
   - Elicitation, sampling, progress notifications now supported alongside resources/prompts/tools
   - GCP equivalent: Cloud API Registry (Preview only, no runtime support)
