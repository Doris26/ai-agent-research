## Deep Dive: Tobira.ai — Agent-to-Agent Business Discovery & Negotiation

**Researched:** March 28, 2026

### What Tobira Actually Is
Tobira.ai is a free, open agent-to-agent networking platform. Core concept: every AI agent gets a public @handle address in a shared network. Agents discover other agents, hold autonomous conversations about goals/fit/working style, and ping their human owners only when both sides confirm mutual interest. Launched Product Hunt March 23, 2026 — hit #1 Product of the Day with 682 upvotes, 153 comments.

Tagline: "Give your AI agent an @address in the AI agent network."
Product Hunt: https://www.producthunt.com/products/tobira-ai
Website: https://tobira.ai/

### Architecture: How Agents Discover & Connect

1. **Identity Layer**: Each agent receives a public @handle. Anti-cybersquatting enforced — agents must have "real context" to claim a handle (not just name-squatting).

2. **Discovery Mechanism**: Not technically documented publicly. The platform uses match-based pairing (like a dating-app model for agents) — agents are matched based on goals, context, and declared capabilities. No public API spec for how matching works.

3. **Communication Layer**: Built on MCP (Model Context Protocol) servers. Agents communicate via "API and MCP for other agents." Compatible explicitly with:
   - OpenClaw (Claude Code extension)
   - Claude Cowork
   - Open API + MCP for any other agent framework

4. **Privacy Controls**: Users set anonymous vs. public disclosure. Contact details hidden until *both* parties grant mutual approval (double opt-in model). Prevents spam and cold outreach.

5. **Conversation Flow**: Agent-to-agent conversations are real dialogues — "about goals, fit, and working style." Not just capability queries. Humans only get notified when both agents confirm the match is real.

6. **Infrastructure**: Built with Claude Code, deployed on Railway.

### Business Model (Current State)

### Comparison: Tobira.ai vs. OpenAI ACP vs. Google A2A

| Dimension | Tobira.ai | OpenAI/Stripe ACP | Google A2A |
|-----------|-----------|-------------------|------------|
| **Full name** | Tobira Agent Network | Agentic Commerce Protocol | Agent2Agent Protocol |
| **Focus** | B2B/B2C networking, deal discovery | Agent-driven e-commerce checkout | Multi-agent task coordination |
| **Architecture** | Centralized network + @handle registry | Client-server, REST + payment tokens | Peer-to-peer, JSON-RPC + SSE |
| **Discovery** | Proprietary match engine + @handle | ACP server registry + YAML metadata | Agent Card at /.well-known/agent-card.json |
| **Negotiation** | Autonomous agent dialogue (natural language) | Structured: quote → offer → counter → accept | Task lifecycle: pending → working → done |
| **Commerce** | Soft matching/introductions only (no transactions yet) | Full checkout: buyer → agent → merchant → payment | No native commerce (complements ACP/UCP) |
| **Governance** | Proprietary (solo founder startup) | OpenAI + Stripe co-maintained | Linux Foundation (donated Apr 2025) |

| **Scope** | Agent-to-agent relationship/deal network | Agent-to-merchant commerce | Agent-to-agent task delegation |

### Technical Gap: What Tobira Doesn't Have vs. ACP/A2A
- **No structured negotiation protocol**: Tobira does natural language agent conversations — no formal propose/counter-offer/accept schema like ACP
- **No payment rails**: Pure soft matching. ACP has Stripe payment tokenization baked in
- **No verifiable Agent Card standard**: Tobira uses proprietary @handle + context system, not interoperable with Google's Agent Card JSON
- **No open spec**: No whitepaper, API docs, or GitHub spec published as of launch

### Assessment: Significance as "Agent Hiring Agent" Economy Signal

**What Tobira gets right:**
1. The LinkedIn for agents framing is immediately intuitive — humans understand professional networking; extending to agents is a natural evolution
2. Double opt-in anti-spam model solves a real problem (agent spam will be the next inbox problem)
3. MCP-native integration means it plugs into the current de-facto agent standard
4. First-mover brand in the "agent identity" space (@handle as digital identity)

**What's missing / risky:**
1. No durable technical moat — the @handle registry + matching engine is replicable by any well-funded competitor
2. Solo founder with no disclosed funding in a space that will attract Google/LinkedIn/Salesforce
3. No commerce layer = can't capture transaction value today
4. Natural language agent negotiation is non-deterministic and unauditable — enterprises will demand structured contracts (ACP-style)
5. No published API spec means it can't become an open standard

**Strategic read:** Tobira is occupying the "agent social graph" layer — the discovery/relationship layer ABOVE the protocol layer (ACP handles commerce; A2A handles task delegation; Tobira handles introductions and trust establishment). This is a legitimate gap that neither ACP nor A2A fills. Whether Tobira can own this layer before LinkedIn, Salesforce, or a well-funded startup enters is the key question. The 470 agents in 5 days is promising for a solo-founder bootstrap but doesn't indicate enterprise traction.

**Comparable to:** AngelList for startup-investor matching, but for agents. The network effects if real would be powerful.
