## Papers Found Today

### Multi-Agent Coordination
- **[2604.01608] From Multi-Agent to Single-Agent: When Is Skill Distillation Beneficial?** — CUHK/LIGHTSPEED. MAS→single-agent distillation lift ranges -2% to +28%. Outcome depends on *metric*, not task. Introduces Metric Freedom (F) as first a priori predictor. Two-stage adaptive distillation framework.
- **[2604.00722] LangMARL: Natural Language Multi-Agent Reinforcement Learning** — ASU/Cisco/UNC. Credit assignment in LLM multi-agent systems via language-space policy gradients. Centralized critic attributes team outcomes to individual agents in natural language. Scales across reasoning, QA, coding, games.
- **[2604.02334] Holos: Scalable LLM Multi-Agent System for Agentic Web** — Five-layer architecture: Nuwa Engine (agent generation), Market-Driven Orchestrator, Endogenous Value Cycle. Web-scale persistent agent ecosystems. Released at holosai.io.

### Tool Use
- **[2604.00835] Agentic Tool Use in Large Language Models** — HIT/TikTok. Three-paradigm taxonomy: prompting → supervised → reward-driven tool learning. Systematizes evaluation from function-call correctness to end-to-end safety/robustness.

### Security (CRITICAL)
- **DeepMind "AI Agent Traps"** (SSRN 6372438) — Franklin et al. First systematic taxonomy of 6 attack categories against autonomous agents: content injection (86% success), semantic manipulation, cognitive state/memory poisoning (80%+ at <0.1% data), sub-agent hijacking (58-90%), human-in-the-loop exploitation. Recommends runtime content scanners, pre-ingestion filters, new web standards for AI-consumable content.

### Safety & Alignment
- **Anthropic A3: Automated Alignment Agent** — Agentic framework for automated safety finetuning. Auto-detects failures via hypothesis testing + data generation, iteratively finetunes. Beats baselines on sycophancy, political bias, jailbreaks. Open-sourced on GitHub.

### Architecture
- **Anthropic Three-Agent Harness** — Separates planning/generation/evaluation agents with context resets + structured handoffs. 5-15 iterations, up to 4 hours per run. Multi-phase orchestration pattern for long-running dev tasks.
