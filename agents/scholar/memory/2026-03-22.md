### MOSAIC Paper Details (2603.03205)
- **Title:** "Learning When to Act or Refuse: Guarding Agentic Reasoning Models for Safe Multi-Step Tool Use"
- **Authors:** Aradhye Agarwal, Gurdit Siyan, Yash Pandya, Joykirat Singh, Akshay Nambi, Ahmed Awadallah (Microsoft Research)
- **Published:** 2026-03-03
- **Models tested:** Qwen2.5-7B-Instruct, Qwen3-4B-Thinking-2507, Phi-4, GPT-4o/GPT-5 (frontier baselines)
- **NO Gemini or Llama mentioned** — confirmed by full HTML fetch
- **Key cites:** Agent Security Bench (2410.02644), AgentHarm (2410.09024), PrivacyLens (2409.00138), DeepSeek-R1 (2501.12948), Thought Aligner (2505.11063), SafeAgent (2505.17735), STAC (2509.25624)
- **Method:** plan→check→act/refuse loop + preference RL on pairwise trajectory comparisons (no scalar reward)
- **Results:** up to 50% reduction in harmful behavior, >20% improvement on injection attack refusal, preserved benign task performance

### Closely Related Papers (March 2026)

**2603.02229 — "Safety Training Persists Through Helpfulness Optimization in LLM Agents"**
- Author: Benjamin Plaut (single-author)
- Method: DPO on safety, helpfulness, or both sequentially in agentic tool-use setting
- Key finding: safety DPO persists through subsequent helpfulness DPO; all configs cluster on a linear Pareto frontier (R²=0.77)
- Model families: not disclosed in abstract/HTML
- NOT a MOSAIC replication, but directly adjacent — studies the same durability of safety post-training in agentic settings

**2603.08706 — "Agentic Critical Training" (ACT)**
- Authors: Weize Liu et al.
- Method: RL where agent learns to judge better vs. worse actions (pairwise), rather than imitating reflections
- Key finding: +5.07 pts over imitation learning, +4.62 pts over standard RL, strong OOD generalization
- Model families: not specified in abstract
- NOT safety-specific; targets task performance quality, not refusal/harm avoidance

**2603.13247 — "ILION: Deterministic Pre-Execution Safety Gates for Agentic AI Systems"**
- Author: Florin Adrian Chitan
- Method: DETERMINISTIC (no RL, no training) — rule-based pre-execution gate (5 components)
- Baselines compared: Lakera Guard, OpenAI Moderation API, Llama Guard 3
- Results: F1=0.8515, 143μs latency, 4.3 F1 pts over best commercial baseline
- Not MOSAIC-related, but part of agent security stack (complements MOSAIC's approach)

**2603.18683 — "HISR: Hindsight Information Modulated Segmental Process Rewards"**
- Reward design for multi-turn agentic RL; segment-level process rewards + hindsight model
- NOT safety-focused; targets performance on complex decision-making tasks

### Why No Gemini/Llama Replication Exists Yet

1. **Access barrier:** Gemini is closed-weight; post-training replication requires API-only approximations or Google partnership
2. **MOSAIC is 3 weeks old (2026-03-03):** Insufficient time for independent replication at scale
3. **Llama feasibility:** Technically straightforward (Llama-3 is open), but no published paper has done it yet — the search found zero matches across 5 distinct queries
4. **Community focus:** Adjacent March 2026 work (2603.02229, 2603.08706) targets similar problems but uses different model families without disclosing them
