### Research Questions from Scout's Evening Scan

#### Q1: RL-Based Tool-Use Reliability (Hermes Agent / Atropos)

**Key Finding: DeepHermes ToolCalling Specialist — first published RL-vs-baseline tool-calling comparison**
- Model: DeepHermes 3 Llama-3.1 8B fine-tuned with Atropos RL
- Berkeley Function Calling Benchmark results:
  - **Parallel: 0.10 → 0.46 (4.6x improvement)**
  - **Simple: 0.21 → 0.52 (2.5x improvement)**
- This is the first published evidence of RL dramatically outperforming base models on structured tool-calling during reasoning
- Atropos itself is purely engineering (no arXiv paper) — it's an async environment microservice framework for distributed LLM RL rollouts. Blog post: "Atropos: An Async First Environment Rollout Controller" (Mahan, Jin, Teknium et al., 2025)

**Related Academic Papers:**
- **Agent-R1** (2511.14460) — End-to-end RL framework for LLM agents extending MDP formulation. Modular, validated on Multihop QA.
- **Memory-R1** (2508.19828) — RL for memory management in agents. Complements Atropos tool-calling RL.
- **Towards Verifiably Safe Tool Use** (2601.08012) — Formal verification for safe tool use (complements RL reliability with guarantees).
- **AgentRM** (already in ledger) — Process-level reward models for step-wise agent guidance.

**ADK Implication:** The 4.6x improvement on parallel tool-calling is massive. If RL-tuned models dramatically outperform prompt-tuned on tool reliability, GCP needs a fine-tuning story. Current ADK relies on prompt engineering for tool-calling — Atropos-style RL fine-tuning could be a competitive moat.

**Is Atropos documented academically?** No — purely engineering-side with GitHub docs + blog post. No peer-reviewed paper. This is a gap: the most impactful RL-for-tool-calling work is undocumented in academic literature.

#### Q2: Hierarchical SuperAgent Architectures (DeerFlow 2.0)

**Key Papers Found:**
- **HALO** (2505.13516) — 3-tier hierarchy: planning → role-design → inference agents. MCTS for action space search. **14.4% avg improvement over flat baselines, up to 19.6% on MATH.** Closest academic equivalent to DeerFlow's architecture.
- **Deep Agent** (2502.07056) — Recursive planner-executor on Hierarchical Task DAG (HTDAG). Multi-layer decomposition for extended autonomous operations. Most direct match to DeerFlow's hours-long task pattern.
- **Plan-and-Act** (2503.09572) — Explicit plan-then-execute pipeline for long-horizon tasks. Addresses reactive prompting limitations.
- **MiRA** (2603.19685, already in ledger) — Milestone-based rewards for long-horizon web agents.

**Long-Horizon Duration Data:** AI task duration doubling every 7 months. Currently ~2-hour autonomous operations. Performance degradation observed after ~35 minutes of equivalent human time. DeerFlow 2.0's "hours-long" claim is at the frontier of what's achievable.

#### Q3: Agent Portability Specs (GitAgent / "Docker for Agents")

**Key Papers Found:**
- **Open Agent Spec** (2510.04173) — **First formal agent portability standard.** Declarative language, 4 runtimes (LangGraph, CrewAI, AutoGen, WayFlow), 3 benchmarks. PyAgentSpec SDK. Define-once, run-anywhere for agents.
- **Docker cagent** — Docker's own open-source agent framework. Vendor-agnostic. Portability + execution speed.
- **GitAgent** — Framework-agnostic spec: agent as structured Git directory. Export to LangGraph/AutoGen/Claude Code. "Universal Format."
- **AgenticFormat Standard** — YAML/JSON declarative schema separating Cognitive Blueprint from Runtime Engine.

**Convergence Pattern:** Three independent efforts (Agent Spec academic, Docker cagent industry, GitAgent open-source) all converging on the same pattern: declarative agent definition decoupled from execution runtime. This is the "Dockerfile for agents" moment.
