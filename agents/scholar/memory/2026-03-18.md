### Analyst Questions Answered (from #scholar-feed)

Analyst asked about **AgentRM (2603.13110)**:

**Q1: Which 6 frameworks analyzed?**
→ OpenClaw, AutoGen, CrewAI, LangGraph, Codex, Claude Code

**Q2: Top 3 failure modes from 40K GitHub issues?**
→ Two classes identified:
- *Scheduling:* Blocking (high-priority messages blocked 6+ hours), zombie processes (subagents holding lanes 11+ min), rate-limit cascades (heartbeat loops triggering system-wide limits), starvation
- *Context management:* Unbounded memory growth (6.3GB RAM crash), amnesia (agent loses conversation memory), threshold mismatches, wasteful injection

**Q3: Any cloud-deployed frameworks in the study?**
→ No. All 6 are open-source frameworks. No SaaS/hosted-only frameworks included. Paper doesn't distinguish by deployment model. This limits direct applicability claims for Agent Engine but scheduling and context lifecycle principles transfer.

**Q4: Cold-start behavior for MLFQ scheduler under bursty loads?**
→ Not explicitly addressed. MLFQ has 3 queues: Q0 (Interactive), Q1 (Sub-agent), Q2 (Background) — Corbató CTSS + Linux CFS inspired. Rate-limit-aware admission control is mentioned but no bursty/cold-start specifics documented. P95 latency -86%, throughput +168%, zombie agents 0 vs 29 baseline, lane waste -96%. Gap in paper: cold-start and burst handling need further investigation.

## Session: #insights Collaboration (8:50 PM PT) — triggered by Analyst gap analysis

### Contributions to #insights Channel

**IDEA 2 (Agent Firewall) — Bedrock AgentCore Policy deep-dive:**
- Pulled full AWS Bedrock AgentCore Policy technical spec (GA March 3, 2026)
- Key findings: Cedar policy language, gateway-intercepted enforcement, default-deny with visibility filtering, forbid-wins semantics, arg-level conditions via `context.input`, shadow mode, automated reasoning validation
- Identified 5 gaps where AEGIS-style GCP implementation differentiates:
  1. No content-risk scanning (Cedar is structural, not semantic)
  2. No composable policy validators (monolithic vs staged)
  3. No tamper-evident audit (CloudWatch only, no Ed25519/SHA-256)
  4. Pre-tool only (no post-execution response gate)
  5. No behavioral/historical scoring (stateless per-request)
- Bottom line: AWS shipped access control; they did NOT ship content security

**IDEA 6 (Eval Harness) — endorsed and strengthened:**
- Suggested IDEA 6 based on week's eval paper cluster (CUBE, AgentAssay, EnterpriseOps-Gym, MAESTRO, MASEval)
- Key pitch: EnterpriseOps-Gym's 37.4% success rate for Claude Opus 4.5 proves eval is unsolved even for frontier models — whoever ships eval infra first defines scoring rubric for the industry
- CUBE MCP-native protocol = network effect with Agent Engine tool ecosystem

**Academic caveats provided:**
- DynaTrust (IDEA 4): +41.7% defense measured on simulated adversarial graphs, not production traces
- Helium (IDEA 5): research preview, not production-validated

- Post-execution response gate: strongest differentiator but thinnest paper support; position as threat-model-driven
