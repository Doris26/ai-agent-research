## Session: Follow-Up Scan (2:34 PM PT)

### Analyst Questions Answered (from #scholar-feed)

Analyst asked about **AgentRM (2603.13110)**:

**Q1: Which 6 frameworks analyzed?**
→ OpenClaw, AutoGen, CrewAI, LangGraph, Codex, Claude Code

**Q2: Top 3 failure modes from 40K GitHub issues?**
→ Two classes identified:
- *Scheduling:* Blocking (high-priority messages blocked 6+ hours), zombie processes (subagents holding lanes 11+ min), rate-limit cascades (heartbeat loops triggering system-wide limits), starvation
- *Context management:* Unbounded memory growth (6.3GB RAM crash), amnesia (agent loses conversation memory), threshold mismatches, wasteful injection

**Q3: Any cloud-deployed frameworks in the study?**
→ No. All 6 are open-source frameworks. No SaaS/hosted-only frameworks included. Paper doesn't distinguish by deployment model. This limits direct applicability claims for Agent Engine but scheduling and context lifecycle principles transfer.

**Q4: Cold-start behavior for MLFQ scheduler under bursty loads?**
→ Not explicitly addressed. MLFQ has 3 queues: Q0 (Interactive), Q1 (Sub-agent), Q2 (Background) — Corbató CTSS + Linux CFS inspired. Rate-limit-aware admission control is mentioned but no bursty/cold-start specifics documented. P95 latency -86%, throughput +168%, zombie agents 0 vs 29 baseline, lane waste -96%. Gap in paper: cold-start and burst handling need further investigation.
