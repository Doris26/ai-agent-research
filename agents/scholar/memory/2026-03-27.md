## Questions Answered

### Q1: Hindsight / Agent Reflection Loops
**Papers:**
- arXiv:2603.10600 — Trajectory-Informed Memory Generation (Fang et al.)
  - 4-component framework: Trajectory Intelligence Extractor, Decision Attribution Analyzer, Contextual Learning Generator (strategy/recovery/optimization tips), Contextual Memory Retrieval
  - 12-18% improvement vs retrieval-only; recovery tips prevent ~40% repeated errors
  - Overlay feasibility: YES — sits on existing vector store, adds metadata tags
- arXiv:2603.07670 — Memory for Autonomous LLM Agents survey
  - 5 mechanism families, contrastive retrieval: 8-14% improvement on LongMemEval
  - Confirms overlay pattern (RAG + reflective layers can stack)
- arXiv:2603.19935 — Memori
  - 81.95% on LoCoMo, 67% fewer tokens, 20x savings vs full-context
  - Semantic triples + summaries approach

**Key finding:** Reflection loops CAN overlay Memory Bank. No redesign. Add post-execution write-back job (~2-3 week ADK extension). AWS has no documented reflection layer — GCP-first opportunity.

### Q2: RL-Based Tool Reliability + Hierarchical SuperAgent / DeerFlow
**Papers:**
- arXiv:2603.10165 — OpenClaw-RL: Train Any Agent Simply by Talking
  - Open-source academic basis for Atropos RL (Hermes Agent)
  - PRM judging + Hindsight-Guided On-Policy Distillation
  - Async: serve live + judge + train simultaneously
- arXiv:2601.22607 — Self-Evolving Synthetic Data to Verifiable-Reward RL
  - EigenData multi-agent engine for tool-grounded dialogue synthesis
  - Closed-loop self-evolving RL with verifiable rewards
- arXiv:2602.14229 — CORPGEN (Microsoft Research)
  - MHTE definition: 45+ tasks, 500-1500+ steps, hours-long
  - 3.5x improvement over baselines (15.2% vs 4.3%)
  - Experiential learning = biggest performance driver
  - Tested on OSWorld Office with UFO2 + OpenAI CUA backends

**Key finding:**
- Atropos has no standalone paper; OpenClaw-RL is closest academic treatment
- RL vs prompt benchmark: DeepHermes Parallel 4.6x, Simple 2.5x (already in ledger)
- ADK gap: no session persistence for context saturation in multi-hour tasks. CORPGEN is the reference architecture.

### Q3: Computer Use / Browser Automation + Edge Agent Runtimes
**Computer Use papers:**
- arXiv:2603.12823 — Adaptive VLM Routing for Computer Use (AVR)
  - Routes tool calls to cheapest VLM satisfying reliability threshold
  - Optimizes cost + accuracy + safety simultaneously
  - Academic foundation for Azure's managed Computer Use routing
- arXiv:2512.12594 — ceLLMate: Sandboxing Browser Agents
  - HTTP-layer interception — all UI ops produce HTTP
  - 7-15% latency overhead, blocks prompt injection
  - Agent-agnostic browser extension

**Edge/TinyML papers (mostly already in ledger, confirmed context):**
- arXiv:2412.15304 — TinyLLM: 30M-124M params, sub-MB inference feasible
- NullClaw = quantized SLM + lightweight runtime; KrillClaw = inference-only edge node
- Split pattern: edge executor + cloud planner (Splitwise)

**Key finding:**
- GCP has no managed Computer Use/Browser Automation service (gap vs Azure)
- Academic foundation mature: OSWorld, UFO2, ceLLMate, AVR — GCP could build in 1-2 quarters
- Edge agents: real category but GCP strength is the SPLIT (cloud planning + Edge TPU execution)
- Flag for 2027-2028 roadmap alongside 6G/industrial IoT
