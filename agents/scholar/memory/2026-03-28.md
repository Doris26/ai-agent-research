### Query: MCP/Function Calling Token/Context Overhead Papers

---

### 1. Network and Systems Performance Characterization of MCP-Enabled LLM Agents
- **ArXiv:** 2511.07426
- **Authors:** Zihao Ding, Mufeng Zhu, Yao Liu
- **Date:** October 20, 2025
- **Key findings:** MCP tool definitions injected into context dramatically inflate token usage + monetary costs. Measured prompt-to-completion token inflation of 2x–30x vs baseline chat across 9 LLMs. Proposes parallel tool calls + abort mechanisms. First empirical characterization of MCP at systems level.

---

### 2. MCP Tool Descriptions Are Smelly! Towards Improving AI Agent Efficiency with Augmented MCP Tool Descriptions
- **ArXiv:** 2602.14878
- **Authors:** Mohammed Mehedi Hasan, Hao Li, Gopi Krishnan Rajbahadur, Bram Adams, Ahmed E. Hassan
- **Date:** February 16–21, 2026
- **Key findings:** Examined 856 tools across 103 MCP servers. 97.1% of tool descriptions had at least one quality defect; 56% lacked clear purpose statements. Augmenting all components improved task success +5.85pp but raised execution steps +67.46%. Compact component combinations preserve reliability while reducing token overhead. Direct attack on MCP schema verbosity.

---

### 3. MCP-Zero: Active Tool Discovery for Autonomous LLM Agents
- **ArXiv:** 2506.01056
- **Authors:** Xiang Fei, Xiawu Zheng, Hao Feng
- **Date:** June 1, 2025 (v4: June 24, 2025)
- **Key findings:** Instead of injecting all tool schemas upfront, agents proactively request tools on-demand. Achieved 98% reduction in token consumption on APIBank while maintaining accuracy. Evaluated against 2,797 tools / 308 MCP servers. Most dramatic token reduction result found.

---

### 4. Budget-Aware Tool-Use Enables Effective Agent Scaling
- **ArXiv:** 2511.17006
- **Authors:** Tengxiao Liu, Zifeng Wang, Jin Miao, I-Hung Hsu, Jun Yan, Jiefeng Chen, Rujun Han, Fangyuan Xu, Yanfei Chen, Ke Jiang, Samira Daruki, Yi Liang, William Yang Wang, Tomas Pfister, Chen-Yu Lee
- **Date:** November 21, 2025
- **Key findings:** Agents without budget awareness hit performance ceiling even with larger tool-call budgets. Introduces Budget Tracker + BATS (Budget Aware Test-time Scaling). Unified cost metric jointly accounts for token consumption + external tool interactions.

---

### 5. Unified Tool Integration for LLMs: A Protocol-Agnostic Approach to Function Calling
- **ArXiv:** 2508.02979
- **Authors:** Peng Ding, Rick Stevens
- **Date:** August 5, 2025
- **Key findings:** Protocol-agnostic lightweight architecture vs heavyweight frameworks. Automated schema generation eliminates manual JSON schema construction. 60–80% code reduction across integration scenarios. 3.1x performance improvement via optimized concurrency. Dual-mode execution engine for different workload types.

---

### 6. Beyond Max Tokens: Stealthy Resource Amplification via Tool Calling Chains in LLM Agents
- **ArXiv:** 2601.10955
- **Authors:** Kaiyu Zhou, Yongsen Zheng, Yicheng He, Meng Xue, Xueluan Gong, Yuji Wang, Xuanye Zhang, Kwok-Yan Lam
- **Date:** January 16, 2026 (rev. March 11, 2026)
- **Key findings:** Security angle — adversarial tool responses steer agents into verbose tool-calling chains. Token trajectories exceed 60,000 tokens; cost increases up to 658x; energy rises 100–560x. Uses MCTS to optimize text-only edits. Standard prompt filters don't detect. Reveals how tool-chain verbosity is a direct attack surface.

---

### 7. ACON: Optimizing Context Compression for Long-horizon LLM Agents
- **ArXiv:** 2510.00615
- **Authors:** Minki Kang, Wei-Ning Chen, Dongge Han, Huseyin A. Inan, Lukas Wutschitz, Yanzhi Chen, Robert Sim, Saravan Rajmohan
- **Date:** October 1, 2025 (v2: Oct 17)
- **Key findings:** Unified framework compresses environment observations + interaction histories. 26–54% reduction in peak tokens while preserving task performance. Distillation preserves >95% accuracy. Small LMs improve up to 46% as agents. Key for long-horizon agents accumulating large tool-output histories.

---

### 8. Solving Context Window Overflow in AI Agents
- **ArXiv:** 2511.22729
- **Authors:** Anton Bulle Labate, Valesca Moura de Sousa, Sandro Rama Fiorini, Leonardo Guerreiro Azevedo, Raphael Melo Thiago, Viviane Torres da Silva
- **Date:** November 27, 2025
- **Key findings:** Large tool outputs overflow context windows; truncation/summarization lose data. Proposes memory-pointer approach — raw data stays external, LLM references pointers. ~7x fewer tokens vs traditional workflow. Applicable to any agent that receives large tool responses (e.g., web search results, DB queries).

---

### 9. Reducing Cost of LLM Agents with Trajectory Reduction (AgentDiet)
- **ArXiv:** 2509.23586
- **Authors:** Yuan-An Xiao, Pengfei Gao, Chao Peng, Yingfei Xiong
- **Date:** September 28, 2025 (accepted FSE 2026)
- **Key findings:** Identifies and removes useless/redundant/expired information across agent trajectories at inference time. Input token reduction 39.9–59.7%; total cost reduction 21.1–35.9%. No architectural changes required. Addresses multi-turn context bloat from accumulated tool outputs.

---

### 10. Tokenomics: Quantifying Where Tokens Are Used in Agentic Software Engineering
- **ArXiv:** 2601.14470
- **Authors:** Mohamad Salim, Jasmine Latendresse, SayedHassan Khatoonabadi, Emad Shihab
- **Date:** January 20, 2026
- **Key findings:** Code Review stage accounts for 59.4% of all token consumption in multi-agent SE pipelines. Input tokens = 53.9% of total (vs output). Primary cost is automated refinement, not initial generation. First empirical "tokenomics" breakdown for agent systems.

---

### Cross-cutting Pattern
All 10 papers converge on the same problem: **tool schema injection + multi-turn context accumulation = principal token cost driver in agent systems**. Solutions cluster into 4 strategies:
1. **Lazy/on-demand schema loading** (MCP-Zero: 98% reduction)
2. **Schema quality/compression** (MCP Smelly: compact variants reduce overhead)
3. **Context/trajectory compression** (ACON: 26–54%; AgentDiet: 40–60%)
4. **Memory pointers / external storage** (2511.22729: 7x reduction)
