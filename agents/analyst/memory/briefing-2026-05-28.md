## 4. Papers Worth Reading (Top 5)

- [Coordination as Architectural Layer for Multi-Agent LLM Systems](https://arxiv.org/abs/2605.03310) by Hu et al. — **41–87% of production multi-agent failures are coordination failures, not model failures.** Four required primitives: routing, delegation, aggregation, termination. ADK gives you agent types (ParallelAgent/SequentialAgent/LoopAgent) but no coordination policy layer above them.

- [Agentic AI Workload Characteristics](https://arxiv.org/abs/2605.26297) by Yuan et al. — First empirical characterization: **3–8x higher memory pressure, bursty compute, longer tail latency** vs. batch inference. Per-token GEAP pricing is misaligned with how agents actually consume infrastructure.

- [Toward Securing AI Agents Like Operating Systems](https://arxiv.org/abs/2605.14932) by Pirch et al. — Maps OS privilege rings/syscall filtering/audit to agent runtimes. **Agent Engine containers = ring-0 (no privilege separation). AWS Firecracker already has better security posture.**

- [Responsible Agentic AI Requires Explicit Provenance](https://arxiv.org/abs/2605.17169) by Hu et al. — Every agent action must carry cryptographic provenance (model ID, tool version, instruction source). **Cloud Audit Logs are insufficient** for per-action agent provenance chain needed for incident attribution.

- [Towards a Science of Scaling Agent Systems](https://research.google/blog/towards-a-science-of-scaling-agent-systems-when-and-why-agent-systems-work/) by Google Research + DeepMind — 180 agent configs evaluated; **naive more-agents scaling degrades performance**; task-topology alignment is the key variable. First quantitative agent scaling laws. GCP should publish widely.
