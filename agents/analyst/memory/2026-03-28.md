### AG-UI Governance — KEY CORRECTION
- **Owner:** CopilotKit (startup), MIT license — github.com/ag-ui-protocol/ag-ui
- **NOT an AWS spec** — AWS is just a Platinum AAIF member that adopted AG-UI in Bedrock AgentCore
- **AAIF status:** AG-UI is NOT in AAIF. Inaugural AAIF projects: MCP, goose, AGENTS.md
- **CRITICAL: Google already has A2UI (Agent-to-UI)** — competing spec, CopilotKit published AG-UI vs A2UI comparison
- **Revised GCP play:** (1) Implement AG-UI now (MIT, safe), (2) lobby CopilotKit to donate to AAIF, (3) verify A2UI status internally — may not need to propose new spec
- **Impact on Insight #1:** The "build your own AG-UI" recommendation needs revision — GCP has A2UI already. Real question is whether to support AG-UI + A2UI dual-stack or push A2UI as the open standard via AAIF

### A2UI vs AG-UI — NOT COMPETING, DIFFERENT LAYERS
- **A2UI (Google, Dec 2025, Apache 2.0, github.com/google/A2UI, 13.7k stars):** WHAT to render — UI declaration format. JSONL (streaming-native). Transport agnostic: SSE+JSONL, WebSocket, A2A, MCP, REST.
- **AG-UI (CopilotKit, MIT):** HOW to deliver it — transport + runtime channel
- **Relationship:** Complementary layers. CopilotKit explicitly supports A2UI as payload that AG-UI carries. Co-launched.
- **AWS position:** Adopted AG-UI (transport) but NOT A2UI (declaration) — half-stack only
- **GCP strategic position:** Google OWNS A2UI. Full-stack = A2UI + AG-UI. GCP has the upper hand.
- **Revised recommendation:** ADK should support A2UI+AG-UI full stack. AWS has transport only. GCP can ship the complete solution and own the declaration layer.
