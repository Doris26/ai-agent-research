## Executive Summary

- **CUA arms race complete:** OpenAI SDK v0.17.4 ships Windows Computer Use — all three major competitors (Anthropic Mac, Azure desktop, OpenAI Windows) now have OS-level CUA tools. GCP is the only major cloud without a named drop-in managed CUA primitive. Gemini 2.5 CU model exists but requires custom wiring — one integration sprint separates GCP from parity.
- **Pricing threat from workflow compilation:** arXiv:2605.22502 shows compiling multi-step orchestration logic into model weights achieves near-frontier quality at **100x lower inference cost**. This challenges the hosted per-token runtime model. GCP must own the compilation pipeline before customers abandon hosted orchestration.
- **Memory Bank time bomb:** Three papers converge on agent lifespan failure (arXiv:2605.26302, 2605.17830, 2605.26252). Memory Bank just went GA with automated event-triggered memory generation — and zero lifespan monitoring, health signals, or refresh mechanisms. Silent degradation is now accumulating in production.
- **AWS widens memory metadata gap:** AgentCore Memory adds custom metadata tags + hybrid retrieval. GEAP Memory Bank remains semantic-search-only — table stakes gap for production memory systems.
- **A2A v1.1 roadmap validated:** CARA framework (arXiv:2605.20548) proves absence of reasoning traces + verification signals in agent messages causes 12-18pp performance degradation — direct scientific basis for typed A2A message schema upgrade.

## Papers Worth Reading (Top 5)

- [Compiling Agentic Workflows into LLM Weights](https://arxiv.org/abs/2605.22502) by Zhang et al. — **PRICING RISK**: Compiles multi-step orchestration into model weights at 100x lower cost. GCP must own this pipeline or customers bypass hosted runtime.

- [Your Agents Are Aging Too: Lifespan Engineering for Deployed AI Systems](https://arxiv.org/abs/2605.26302) by Multiple — **URGENT**: Memory accumulation, context drift, tool schema drift confirmed as primary lifespan failure modes. Memory Bank has zero lifespan monitoring.

- [What Do Agents Communicate? CARA Framework](https://arxiv.org/abs/2605.20548) by Multiple — Reasoning traces + verification signals absence in agent messages = 12-18pp performance loss. Directly maps to A2A v1.1 typed schema design.

- [AgentFugue: Agent Scaling via Collective Reasoning](https://arxiv.org/abs/2605.24486) by Multiple — Shared collective notes for parallel agents yield 8-24% improvement. ADK parallel subagents have no peer shared state primitive.

- [Why Does Agentic Safety Fail to Generalize Across Tasks?](https://arxiv.org/abs/2605.06992) by Multiple — Formal proof safety training on domain A doesn't transfer to domain B. **Model Armor's single static filter is architecturally insufficient.**
